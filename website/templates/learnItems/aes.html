AES Origin

AES (Advanced Encryption Standard), also known as Rijndael, is a symmetric block cipher cryptosystem, which was standardized by the NIST in 2001. The system was developed by two Belgian cryptographers, Vincent Rijmen and Joan Daemen, and the name Rijndael is a contraction of their names.

The block ciphers are schemes for encryption or decryption where a block of plaintext is treated as a single block and is used to obtain a block of ciphertext with the same size. Today, AES is one of the most used algorithms for block encryption. The size of an AES block is 128 bits, whereas the size of the encryption key can be 128, 192 or 256 bits. Please note this, there is three length in the key, but the size of the encryption block always is 128 bits.

Five modes: ECB, CBC, CFB, OFB, CTR
In this case we are discussing the CTR (short for counter) mode. At this mode of operation as an input block to the encryptor, i.e. as an IV, the value of a counter (Counter, Counter + 1,â€¦, Counter + N - 1) is used. It also is a stream encryptor.

Pros:
- No padding
- Supports parallel computing
- Encryption and decryption use the same structure
- Bad block only affect the current block
Cons:
- A MITM (Man in the Middle) can change some ciphertext damaged plaintext

AES-CTR Description

A secret symmetric key is generated or provided. The AES algorithm takes this key and expands it into a set of round keys that will be used in the subsequent encryption process.

An Initialization Vector is a unique value that is required for the CTR mode. The IV can be generated randomly or derived from a nonce (number used once) and a counter. The IV should be unique for each encryption operation to maintain security. The counter value is initialized based on the IV. The counter is a unique value that is incremented for each block of plaintext or ciphertext.

AES-CTR operates by dividing the plaintext into fixed-size blocks (e.g., 128 bits). For each block, the counter value is encrypted using the AES algorithm and the round keys generated in the key expansion phase. This encryption of the counter produces a pseudorandom keystream.

The pseudorandom keystream generated from the encryption step is then XORed with the corresponding plaintext block. The result of the XOR operation is the ciphertext block. This process is repeated for each block of plaintext, generating the complete ciphertext.

After each encryption or decryption operation, the counter value is incremented to ensure that a unique value is used for each block.
By using this process, AES-CTR transforms the AES block cipher into a stream cipher, generating a pseudorandom keystream that is combined with the plaintext or ciphertext through the XOR operation.

AES-CTR Security

To ensure security, the key in this mode need to be changed for every 2^(n/2) encryption blocks.

Attack modes:
- PA (Padding Attack)
- CPA (Chose Plaintext Attack)
- CCA (Choosen Ci)

It's commonly useid in:
- Network Communication
- Disk and File Encryption
- Cloud Storage
- Real-Time Media Encryption
- Wireless Security
